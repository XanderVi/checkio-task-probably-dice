For this task, you're about to play a game, but first you need to prepare. The game itself is very simple. Both players roll a single die and whoever rolls highest scores a point. (On a tie, both players reroll.)

But these aren't standard dice. Each player can put any positive number on any side of the die as long as the number of sides match and the total of all the chosen numbers are the same. For example, one player might use a six sided die with the numbers [3, 3, 3, 3, 6, 6] while the other player uses a six sided die with the numbers [4, 4, 4, 4, 4, 4]. The interesting part of this game is that even with the same number of sides and the same total, different dice have different chances of winning. Using the example dice, the player with all 4's will win 2/3 of the time.

To prepare for this game, you're investigating different ways of picking the numbers. To do this, write a program that will take an opponent's die and output some die that will win against it more than half the time.

Input: A sorted list of integers, one for each face of the opponent's die.

Output: A list of integers, one for each face of your die. If no die satisfies the task requirements, return an empty list.

Example: 

checkio([3, 3, 3, 3, 6, 6]) == [4, 4, 4, 4, 4, 4] # Or [3, 3, 4, 4, 5, 5]
checkio([4, 4, 4, 4, 4, 4]) == [2, 2, 5, 5, 5, 5] # Or [5, 5, 2, 2, 5, 5]
checkio([2, 2, 5, 5, 5, 5]) == [3, 3, 3, 3, 6, 6]
checkio([1, 1, 3]) == [1, 2, 2]
checkio([1, 2, 3, 4, 5, 6]) == [] # Any 6-sided die totaling 21 has a 50/50 chance of winning against the standard die.
checkio([2, 3, 4, 5, 6, 7]) == [1, 1, 3, 7, 7, 8] # This can be beat though.

How it is used: This task illustrates some of the unintuitive results in probability. Many people would think that two dice as similar as those in the task description (same number of sides, same average roll) would have a roughly 50/50 chance against each other. Even more unusual is that sets of dice can be created where die A beats die B, die B beats die C and yet die C beats die A (e.g. the first three examples above). These are called nontransitive dice.

Preconditions:
3 ≤ len(die) ≤ 10
sum(die) ≤ 100
min(die) > 0
